<p>先说明一下，这个电台是纯音乐电台，没有画面。要搞画面的话，建议了解 <code>nginx-module-rtmp</code>。</p>
<p>开一个电台，需要两个部分：电台推流软件，电台服务器，以及收听软件。就像传统的电台一样，得有个录音室，发射塔，然后是收音机。啊哈，是不是回到了各位童年，听着中国之声呢(不是)。</p>
<h2 id="电台服务器icecast">电台服务器～Icecast</h2>
<p>实际上我接触过两个开电台的软件，一个是上面提到的 <code>nginx-module-rtmp</code>。不过这玩意更像是视频服务器，我就不想说了。</p>
<p><a class="link" href="https://www.icecast.org"  target="_blank" rel="noopener"
    >Icecast</a> 是一个音乐电台服务器，也就是说，它接受电台推流软件传来的数据，经过处理(包装)，然后向收听软件推送数据。这里我就不说啥 <code>m3u8</code>，<code>ogg</code>是啥了，毕竟我也不知道，而且又不影响咱用，对不对啊:-)</p>
<p>既然 Icecast 是一个服务器软件，那么<del>它就得运行在服务器上</del>实际上是个电脑就能运行，不过最好还是个服务器吧，比如说你在网上买到的阿里云服务器之类。不过如果你只是想在你的家里搞个自嗨广播，那电脑直接运行也好。前提是你能处理好路由器端口映射和电脑的防火墙，那就不是我的事情了2333</p>
<p>以下我用 Linux 系统举例子，更特殊的说，是 Debian 11 。其他的发行版，应该能举一反三吧&hellip;&hellip;.</p>

<h2 id="电台推流软件三例">电台推流软件三例</h2>

<h3 id="obs-还能推流广播">OBS 还能推流广播？</h3>
<p>你没看错，OBS 也能推流到 Icecast 服务器上，不过我觉得没有那么方便了。毕竟他只是个视频直播软件啊&hellip;&hellip;</p>
<p>首先说点闲话，Icecast 能处理视频流，就是你们直播到 Bilibili 的那个。详情请看<a class="link" href="https://epir.at/2018/03/08/obs-icecast-streaming/"  target="_blank" rel="noopener"
    >这个链接</a>。不过这玩意不是官方支持，所以我不会过多描述，但是我以下的配置文件是根据这玩意改的。</p>
<p>打开 OBS 的设置界面，调到 输出 -&gt; 录像 ，类型选择 &ldquo;自定义输出 FFMpeg&rdquo; 上面。然后咱这么修改：</p>
<ol>
<li>
<p>FFMpeg 输出类型改成&quot;输出到 URL&quot;，下面的 URL 改成 <code>icecast://source:上面设置的推流密码@服务器ip或者域名:8000/stream</code></p>
</li>
<li>
<p>容器格式选择 opus(音频)，下面的混流器设置填上 <code>content_type=application/ogg</code></p>
</li>
<li>
<p>下面的音频比特率填个合适的，比如 192kbps 之类，想起了下载 MP3 年代了吗？</p>
</li>
</ol>

<h3 id="mixxx-感觉更适合些">Mixxx 感觉更适合些</h3>
<p>Mixxx 是一个 DJ 软件，他能混音各种各样的音乐，也能按照顺序播放音乐&hellip;&hellip;好吧，我对这个软件没有那么了解，只是知道这个东西可以用来广播:-P</p>
<p>首先，你最好有个歌库啥的，也就是说，你的电脑得有一堆歌曲文件。这玩意下载也没那么难吧，随便开个网易云，腾讯啥的，一堆可以下载。把他们放在一起，然后在软件设置里面规定好歌库位置。等待然后在音轨选项里面全选之，右键选择&quot;放到自动DJ&quot;。打开自动DJ界面，点击启用自动Dj按钮，好了，广播台现在能循环你的歌单了。歌单还能随机播放哦。</p>

<h3 id="ffmpeg-极简主义">FFMpeg 极简主义</h3>
<p>没错，ffmpeg 也能推流 icecast 捏。不过一般都是用来转播别的广播台233，图个测试用途。看看我用来推流中国之声到服务器上面的命令。</p>
<div class="highlight"><pre tabindex="0" class="chroma"><code class="language-bash" data-lang="bash"><span class="line"><span class="cl">ffmpeg -re -i <span class="s2">&#34;https://ngcdn001.cnr.cn/live/zgzs/index.m3u8&#34;</span> <span class="se">\ </span>   <span class="c1"># 音频源头</span>
</span></span><span class="line"><span class="cl">       -f opus -content_type application/ogg <span class="se">\ </span>   <span class="c1"># 推流的格式和发送过去的 Content_Type</span>
</span></span><span class="line"><span class="cl">       -ice_description <span class="s2">&#34;转播中国之声&#34;</span> -ice_name <span class="s2">&#34;CNR News Transfer&#34;</span> <span class="se">\ </span><span class="c1"># 该推流的描述和名称(不填写也可以)</span>
</span></span><span class="line"><span class="cl">       <span class="s2">&#34;icecast://source:上面设置的推流密码@服务器ip或者域名:8000/cnr-news-transfer&#34;</span> <span class="c1"># 推流目的地址 </span>
</span></span></code></pre></div><p>具体想知道咋用的话，看看<a class="link" href="https://ffmpeg.org/ffmpeg-protocols.html#Icecast"  target="_blank" rel="noopener"
    >这个链接</a>。</p>
<p>最后说一句，本人十分不建议推流视频到 Icecast 服务器上，挺难用的。( Icecast 官方说支持 opus 和 webm，我知道最后那个是视频格式 )</p>
